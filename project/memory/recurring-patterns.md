# Recurring Patterns

Use this page for repeated project patterns, recurring pitfalls, and stable local tactics that keep paying off inside this project.
Keep entries de-duplicated and project-specific.

This starter page should begin empty in a copied project.
Add entries only after a pattern has repeated or clearly become durable.

## Include here

- repeated pitfalls that future work should avoid
- recurring decision triggers that keep narrowing the right path
- stable local tactics that have worked more than once in this project

## Do not include here

- generic workflows that should become skills
- current task notes
- validation transcripts
- time-ordered session history

## Entry format

- **Pattern:** short name
  When it shows up: repeat trigger or situation
  What to do: short project-local response
  Links: optional pointer to a decision, fact, or active spec
