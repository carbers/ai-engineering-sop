# Working Agreements

Use this page for long-lived project-specific agreements, constraints, preferences, and standing reminders.
Keep entries short, merge duplicates, and update wording in place when the agreement evolves.

This starter page should begin empty in a copied project.
Add entries only when the agreement is specific to that project and likely to matter again.

## Include here

- naming or layering conventions that keep repeating
- collaboration rules that should stay visible across tasks
- environment assumptions or standing constraints
- reminders that repeatedly shape implementation choices

## Do not include here

- one-off task choices
- frozen decisions with full rationale
- task checklists or work logs
- temporary debugging notes

## Entry format

- **Agreement:** one-sentence rule or reminder
  Why it matters: short project-facing reason
  Links: optional pointer to a decision, fact, or active spec
