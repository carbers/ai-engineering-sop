# Experiments

Use this directory when the project starts running experiments that need readable summaries and durable comparison points.

Do not create experiment records for normal task implementation.
Use this area when the work includes explicit experiment setup, results, and follow-up decisions worth re-reading later.

## Naming

Store each experiment in a dated directory:

`EXP-YYYYMMDD-NNN/`

Use `README.md` inside that directory for the readable summary.

## Typical content

- goal
- inputs
- what changed
- headline result
- key metrics
- follow-up decision
